# Hermit i18n Design

Hermit’s current i18n goal is not “translate every string at once.” The immediate goal is to make user-facing copy maintainable and safely fallback-capable through a catalog.

## Current Coverage

The main areas already integrated are:

- top-level CLI help text
- menu bar companion copy
- plugin descriptions and variable descriptions in `plugin.toml`

What is explicitly not translated:

- command names
- tool names
- schema field names
- hook names
- slash command names
- protocol fields that directly affect model behavior

## Locale Resolution Order

Current resolution order:

1. `HERMIT_LOCALE`
2. `LC_ALL`
3. `LC_MESSAGES`
4. `LANG`
5. default `en-US`

The implementation lives in [`src/hermit/infra/system/i18n.py`](https://github.com/heggria/Hermit/blob/main/src/hermit/infra/system/i18n.py).

## Normalization Rules

Current aliases:

- `en` -> `en-US`
- `en-us` -> `en-US`
- `zh` -> `zh-CN`
- `zh-cn` -> `zh-CN`
- `zh-hans` -> `zh-CN`

For a general `lang-region` shape, normalization is:

- lowercase language
- uppercase region

For example:

- `fr-fr` -> `fr-FR`

## Catalog Structure

Current catalog files:

- `src/hermit/infra/system/locales/en-US.json`
- `src/hermit/infra/system/locales/zh-CN.json`

The format uses flat keys:

```json
{
  "cli.app.help": "Hermit personal AI agent CLI.",
  "menubar.title": "Hermit",
  "plugin.scheduler.description": "Scheduled task execution..."
}
```

Do not use English source sentences as keys. That would couple code references to future copy changes.

## How the Code Uses It

At the Python layer, values are resolved through `tr()`:

```python
from hermit.i18n import tr

text = tr("cli.app.help", locale="zh-CN")
```

When a key is missing:

- fallback first goes to `default=...`
- then to the key itself

This ensures that missing catalog entries do not block program startup.

## How `plugin.toml` Localization Works

Plugin descriptions support:

- `description_key`
- `description`

Example:

```toml
[plugin]
name = "usage"
description_key = "plugin.usage.description"
description = "显示当前会话的 token 消耗统计"
```

Meaning:

- if a localized catalog is available, `description_key` wins
- otherwise Hermit falls back to `description`

Variable descriptions follow the same rule.

## Current Boundaries

i18n currently only covers the human-facing copy layer. It has not been extended into:

- the agent system prompt protocol layer
- the model-facing description layer in tool schemas
- full internationalization of Feishu card templates and business output

This is intentional, to avoid destabilizing model behavior just to translate UI copy.

## Highest-Value Next Areas

The next most valuable places to extend coverage are:

- help and error text for CLI subcommands
- Feishu outbound copy
- the human-facing part of `ToolSpec.description`
